Lab Notes — Protocols & Ports
UDPUser Datagram Protocol
UDP is faster but less reliable than TCP. Used for streaming media, DNS queries, VoIP, and real-time applications where speed matters more than guaranteed delivery.
TCPTransmission Control Protocol
TCP is more reliable than UDP. Uses handshaking to guarantee delivery. HTTP (port 80) and HTTPS (port 443) use TCP. Required for web browsing and file transfers.

🔥 1. Limitations of Firewalls
LimitationProblemReal-World Example
Insider ThreatsRegulate external access only; cannot prevent internal attacksMalicious employee exfiltrates data from inside the perimeter
APTsNot designed to detect multi-stage, stealthy intrusionsAttacker uses stolen credentials, blends with normal traffic for months
Encrypted TrafficCannot inspect HTTPS/VPN content without SSL decryptionMalware communicates over HTTPS, bypassing content inspection
False SecurityOver-reliance leaves other vulnerabilities unaddressedMisconfigured rule opens unintended port; weak passwords exploited
Zero-DayDepends on static rules; cannot block unknown attack methodsNovel exploit not yet in vendor signatures bypasses all rules
🔍 2. Limitations of Intrusion Detection Systems
LimitationProblemImpact
False PositivesLegitimate activity flagged as suspiciousAlert fatigue — real threats ignored amid noise
Passive OnlyDetects and alerts; cannot block threats in real-timeDDoS generates alerts but network still saturated
EvasionPacket fragmentation, obfuscation, polymorphic malwarePayload split into harmless fragments; only malicious when reassembled
Encrypted TrafficCannot inspect TLS contentC2 server communication over HTTPS goes undetected
Zero-DaySignature-based detection misses unknown threatsNovel web server exploit bypasses all signature rules
Resource IntensiveNIDS requires heavy resources at high traffic volumesBusy data center NIDS drops packets, creating blind spots

📚 Key Terms Glossary
TermDefinition
FirewallNetwork security device that filters traffic based on predefined rules
IDSIntrusion Detection System — monitors traffic, alerts on suspicious activity (passive)
IPSIntrusion Prevention System — monitors AND actively blocks suspicious traffic
APTAdvanced Persistent Threat — stealthy, long-term attack maintaining unauthorized access
Zero-Day ExploitAttack targeting a vulnerability unknown to the software vendor
False PositiveLegitimate activity incorrectly flagged as malicious
Alert FatigueDesensitization to security alerts due to excessive false positives
Data ExfiltrationUnauthorized transfer of data from an organization's systems
WAFWeb Application Firewall — specialized firewall for web applications (SQLi, XSS)
SIEMSecurity Information and Event Management — aggregates and correlates security data
UEBAUser and Entity Behavior Analytics — ML-based anomaly detection
SSL/TLS InspectionDecrypting and inspecting encrypted traffic to detect hidden threats
