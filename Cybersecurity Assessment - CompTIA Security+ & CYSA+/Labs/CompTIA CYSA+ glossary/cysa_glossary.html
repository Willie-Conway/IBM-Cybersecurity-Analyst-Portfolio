DOMAIN 01
Security Operations
34 TERMS
APT
Advanced Persistent Threat
APT
►
Definition
A type of cyber-attack where a threat actor gains unauthorized access to a network and remains undetected for an extended period, systematically exfiltrating data or preparing for a disruptive payload.
Example
A state-sponsored group infiltrates a government agency's network and remains undetected for months while gathering sensitive intelligence.
INT
Closed Source Intelligence
CLASSIFIED · RESTRICTED
►
Definition
Collects and analyzes classified or restricted information primarily used by government entities. Unlike OSINT, access is controlled and requires appropriate clearances.
Example
A government agency gathers sensitive information on potential threats from foreign adversaries to safeguard national security.
CASB
Cloud Access Security Broker
CASB
►
Definition
A solution that acts as a gatekeeper between users and cloud services, providing visibility, control, and data protection. Enforces security policies across cloud-based apps.
Example
An organization deploys a third-party CASB to monitor and manage user access to cloud applications, ensuring security policies are enforced and sensitive data is protected.
DLP
Data Loss Prevention
DLP
►
Definition
A security strategy that prevents unauthorized disclosure of sensitive data. DLP tools scan content in motion, at rest, and in use to detect and block policy violations.
Example
Software scans outgoing emails for credit card or SSN patterns and automatically blocks or encrypts content before it leaves the network perimeter.
DMARC
DMARC
DOMAIN-BASED MSG AUTH, REPORTING & CONFORMANCE
►
Definition
An email authentication protocol that prevents email spoofing and phishing by verifying sender identity and aligning DKIM and SPF policies. Reports unauthorized use of a domain.
Example
A company configures DMARC policies aligned with DKIM and SPF settings, preventing unauthorized use of their domain to send fraudulent emails.
DKIM
DomainKeys Identified Mail
DKIM
►
Definition
A security method allowing email senders to digitally sign their emails using a private key. Recipients verify authenticity via a public key published in DNS, detecting tampering in transit.
Example
An organization signs all outgoing emails so recipients can confirm the email came from an authorized sender and was not altered during transit.
EDR
Endpoint Detection & Response
EDR
►
Definition
A security solution that continuously monitors endpoints — computers, servers, and mobile devices — to detect, investigate, and respond to suspicious activity and potential threats in real time.
Example
Software deployed on employee computers monitors for unusual activity such as unauthorized file access attempts or system setting changes and triggers alerts automatically.
FED
Federation
IDENTITY FEDERATION
►
Definition
Links multiple identity management systems across organizations, enabling secure access to various applications and services using a single set of credentials via trust agreements.
Example
A user from one organization accesses an application hosted by a partner organization using their existing credentials, without creating a new account.
HAK
Hacktivists
THREAT ACTOR TYPE
►
Definition
Individuals or groups who use hacking techniques to promote a social or political cause. Motivated by ideology rather than financial gain; targets are chosen for symbolic impact.
Example
A group hacks into a corporation's website and defaces it to protest its unethical business practices, drawing public attention.
#
Hashing
CRYPTOGRAPHIC FUNCTION
►
Definition
A one-way mathematical process transforming data of any size into a fixed-length output (hash value). Cannot be reversed; used for integrity checks, password storage, and digital signatures.
Example
A website stores passwords in hashed form (e.g., bcrypt, SHA-256) so a database breach exposes hashes, not plaintext passwords.
HNY
Honeypot
DECEPTION TECHNOLOGY
►
Definition
A decoy system or network resource intentionally deployed to lure attackers. Collects intelligence on attacker tactics, techniques, and procedures (TTPs) without risk to production assets.
Example
A company deploys a decoy server with intentionally vulnerable software, then monitors all interaction with it to map attacker behavior.
IoC
Indicators of Compromise
IoC
►
Definition
Artifacts or patterns — such as IP addresses, file hashes, registry keys, or unusual traffic — that suggest a possible or active security breach or malicious activity within a system or network.
Example
A security analyst detects unusual outbound network traffic to a known malicious IP address, indicating a possible malware infection or C2 beaconing.
INS
Insider Threat
THREAT ACTOR TYPE
►
Definition
A security risk posed by individuals within an organization — employees, contractors, or partners — who have authorized access and misuse it, whether maliciously or through negligence.
Example
An employee with authorized access to confidential data deliberately leaks it to a competitor for financial gain.
INTL
Intelligence Gathering
RECONNAISSANCE
►
Definition
The collection of information about a target system or organization to identify potential attack vectors, vulnerabilities, key personnel, and infrastructure prior to an attack or assessment.
Example
A threat actor researches publicly available information about a company's network infrastructure and employees to prepare a targeted spear-phishing campaign.
PEN
Internal Network Pen Test
PENETRATION TESTING
►
Definition
A test that simulates behavior of malicious insiders or individuals with stolen credentials, seeking to exploit vulnerabilities from within the network perimeter.
Example
A company hires ethical hackers to attempt accessing sensitive systems using stolen credentials and insider knowledge to find lateral movement paths.
LOG
Log Analysis
FORENSICS · MONITORING
►
Definition
Collecting and systematically reviewing log data from network devices, servers, and applications to identify anomalies, security events, performance issues, or evidence of compromise.
Example
A security team examines web server access logs to identify brute-force login attempts, suspicious user agents, or SQL injection patterns.
MFA
Multifactor Authentication
MFA
►
Definition
An authentication method requiring users to verify identity with two or more independent factors: something you know (password), have (token/phone), or are (biometric).
Example
A user enters their password, then a time-based one-time code sent to their mobile device, before being granted access to a cloud application.
NSA
Nation-State Actor
THREAT ACTOR TYPE
►
Definition
A threat entity sponsored or directly controlled by a government, typically engaged in cyber espionage, intelligence collection, sabotage, or disruption of adversary operations. Highly resourced and sophisticated.
Example
A foreign government conducts sustained cyber-operations to steal classified defense technology from another nation's aerospace contractors.
OSINT
Open-Source Intelligence
OSINT
►
Definition
Collecting and analyzing publicly available data from internet sources — social media, forums, WHOIS records, job postings, and more — to gain actionable insights about targets or threats.
Example
A security analyst uses Shodan, LinkedIn, and Maltego to map a target organization's exposed infrastructure and employee structure.
OTI
Operational Threat Intelligence
THREAT INTEL · OPERATIONAL
►
Definition
Intelligence focused on current and emerging risks that could directly impact day-to-day organizational operations. Actionable data about ongoing campaigns, active threat actors, and live attack vectors.
Example
An organization monitors threat feeds for active ransomware campaigns targeting their specific industry vertical to proactively patch affected systems.
ORG
Organized Crime
THREAT ACTOR TYPE
►
Definition
Coordinated criminal groups leveraging cyber capabilities for financial gain through fraud, ransomware, data theft, and extortion. Often operate as professional enterprises with hierarchical structure.
Example
A cybercriminal syndicate breaches a retailer's network, exfiltrates millions of payment card records, and sells them on dark web marketplaces.
PCAP
Packet Capture
PCAP · PACKET SNIFFING
►
Definition
Intercepting and recording raw network traffic for subsequent analysis, troubleshooting, or forensic investigation. Tools like Wireshark capture packets at the network interface level.
Example
A network administrator uses Wireshark to capture and inspect all traffic on a segment to identify malicious C2 communications or data exfiltration patterns.
PII
Personally Identifiable Information
PII
►
Definition
Any data that can identify an individual — name, SSN, DOB, address, biometrics, or account numbers. Protected by regulations such as GDPR, HIPAA, and CCPA.
Example
A customer submits their full name, address, and payment card details during an online purchase — all of which constitute PII requiring protection.
PAM
Privileged Access Management
PAM
►
Definition
A security system that controls, monitors, and audits access to privileged accounts (administrator, root, service accounts). Enforces least-privilege, just-in-time access, and session recording.
Example
An IT admin requests temporary elevated access via a PAM vault, which verifies their identity with MFA, grants time-limited credentials, and records the entire session.
PKI
Public Key Infrastructure
PKI
►
Definition
A system of hardware, software, policies, and procedures that manages digital certificates and public-private key pairs, enabling secure communications, authentication, and non-repudiation.
Example
A website uses a TLS certificate issued by a CA (Certificate Authority) to establish an encrypted HTTPS session between the browser and server.
RISK
Risk Management
GRC · GOVERNANCE
►
Definition
The process of identifying, assessing, and prioritizing potential threats to organizational assets, followed by coordinated application of controls to minimize, monitor, and manage risk probability and impact.
Example
A security team conducts a risk assessment, identifies critical asset exposures, and implements mitigating controls prioritized by risk score (likelihood × impact).
SBX
Sandboxing
ISOLATION TECHNIQUE
►
Definition
A security technique that isolates applications or code execution in a restricted environment, preventing potentially malicious programs from affecting the host system or network.
Example
A suspicious email attachment is opened in an isolated VM environment and monitored for malicious behavior — network calls, file writes, registry changes — before delivery to the user.
SK
Script Kiddie
THREAT ACTOR TYPE
►
Definition
An unskilled attacker who uses pre-written hacking tools and exploit scripts without understanding the underlying technology. Relies on automated tools and publicly available exploits.
Example
An individual downloads a LOIC (Low Orbit Ion Cannon) tool from GitHub and uses it to launch a DDoS attack without understanding TCP/IP or network protocols.
SASE
Secure Access Secure Edge
SASE
►
Definition
A cloud-based architecture converging network security and WAN capabilities into a single service. Combines SWG, CASB, FWaaS, ZTNA, and SD-WAN to secure distributed workforces.
Example
An enterprise replaces regional firewalls and backhauled VPNs with a SASE platform, applying consistent security policy to remote users accessing cloud apps.
SPF
Sender Policy Framework
SPF
►
Definition
An email validation technique that allows receiving mail servers to verify whether a sending server is authorized to send email on behalf of a given domain, using DNS TXT records.
Example
A company publishes an SPF record in DNS designating only their authorized mail servers, causing spoofed emails from unauthorized IPs to fail authentication checks.
SSO
Single Sign-On
SSO
►
Definition
An access control method allowing users to authenticate once with a single credential set and gain access to multiple systems or applications without re-authenticating for each one.
Example
An employee logs into their corporate portal once and is automatically authenticated to email, HR systems, and collaboration tools throughout the session.
SDN
Software-Defined Networking
SDN
►
Definition
A network architecture that separates the control plane from the data plane, allowing network behavior to be programmatically controlled via APIs. Enables dynamic, policy-driven network management.
Example
A data center uses SDN to dynamically segment traffic during an incident, isolating a compromised host without physically reconfiguring switches.
STI
Strategic Threat Intelligence
THREAT INTEL · STRATEGIC
►
Definition
High-level intelligence focused on identifying long-term threats and macro trends that could impact an organization's strategic goals. Consumed by executives and board-level stakeholders.
Example
A CISO presents the board with a strategic threat assessment covering nation-state risks, regulatory shifts, and geopolitical cyber trends affecting the company's 3-year security roadmap.
TTI
Tactical Threat Intelligence
THREAT INTEL · TACTICAL
►
Definition
Intelligence focused on specific, near-term threats and the TTPs (tactics, techniques, procedures) used by threat actors, used to inform defensive configurations and incident response playbooks.
Example
A SOC team analyzes MITRE ATT&CK patterns from recent attacks on peer organizations to harden detection rules and update firewall block lists.
TECH
Technical Threat Intelligence
THREAT INTEL · TECHNICAL
►
Definition
Detailed technical data about specific threats: malware hashes, C2 IPs, exploit signatures, vulnerability CVEs. Consumed by security engineers for tool configuration and defensive tuning.
Example
An analyst reverse-engineers a malware sample to extract C2 domain IoCs and YARA signatures, then pushes them to EDR and SIEM platforms for automated detection.
TA
Threat Actor
ADVERSARY CLASSIFICATION
►
Definition
An individual or group responsible for executing cyber attacks. Classified by motivation (financial, political, ideological), capability level, and attribution (nation-state, criminal, hacktivist, insider).
Example
A financially-motivated threat actor targets a healthcare provider with ransomware, encrypting patient data and demanding payment for the decryption key.
HUNT
Threat Hunting
PROACTIVE DETECTION
►
Definition
A proactive, hypothesis-driven approach to finding threats already present in an environment before alerts are triggered. Uses analytics, behavioral baselines, and TTPs to discover stealthy attackers.
Example
A threat hunter forms a hypothesis that an attacker may be using living-off-the-land techniques, then queries SIEM data for anomalous PowerShell execution patterns across the fleet.
TI
Threat Intelligence
CTI · CYBER THREAT INTEL
►
Definition
Evidence-based knowledge about existing or emerging threats — including context, mechanisms, indicators, and actionable advice — that helps organizations understand and defend against attacks.
Example
A company subscribes to a commercial CTI feed that provides real-time IoCs, threat actor profiles, and vulnerability intelligence to feed into their SIEM and SOAR platforms.
UBA
User Behavior Analysis
UBA / UEBA
►
Definition
A security approach using machine learning and analytics to baseline normal user behavior and detect deviations that may indicate insider threats, account compromise, or privilege abuse.
Example
A UEBA platform detects that an employee is accessing sensitive finance files at 2AM from an unfamiliar location and triggers an alert for SOC investigation.
DOMAIN 02
Vulnerability Management
13 TERMS
ACT
Active Scanning
VULN ASSESSMENT
►
Definition
The process of actively probing and interacting with target systems — sending packets, testing services, and enumerating responses — to uncover exploitable vulnerabilities and misconfigurations.
Example
A penetration tester runs Nessus or OpenVAS against a network segment to enumerate open ports, running services, outdated software versions, and common CVEs.
ADV
Adversary Emulation
RED TEAM · PURPLE TEAM
►
Definition
Replicating the exact TTPs of specific, known threat actors to evaluate how well security controls detect and respond to realistic attack scenarios. More targeted than generic penetration testing.
Example
A red team executes an adversary emulation exercise based on APT29's documented techniques from the MITRE ATT&CK framework to evaluate the organization's detection coverage.
AGT
Agent Scanning
HOST-BASED ASSESSMENT
►
Definition
A vulnerability assessment method using lightweight software agents installed on endpoints to perform continuous, authenticated scanning and report local security posture, patches, and configuration status.
Example
A security team deploys Tenable Nessus Agents on all workstations to provide real-time visibility into missing patches and configuration drift without requiring network-based scanning credentials.
AGTL
Agentless Scanning
NETWORK-BASED ASSESSMENT
►
Definition
Vulnerability scanning performed over the network without requiring software agents on target systems. Uses network protocols, SNMP, SSH, or WMI to remotely query system state.
Example
A company uses agentless scanning to assess cloud infrastructure and IoT devices where installing agents is impractical, using cloud provider APIs instead.
ASM
Attack Surface Management
ASM · EASM
►
Definition
Continuous identification, inventory, and reduction of all externally and internally accessible assets that could be exploited. Includes unknown assets, shadow IT, and digital supply chain components.
Example
A security team uses an ASM platform to discover a forgotten developer portal exposed to the internet with outdated Apache and default credentials.
AUTH
Authentication
IDENTITY VERIFICATION
►
Definition
The process of verifying that an entity (user, device, or process) is who or what it claims to be, before granting access to systems or resources. Distinct from authorization (what access is granted).
Example
A web application challenges users with username + password (knowledge factor) plus a biometric scan (inherence factor) before granting access — multi-factor authentication.
BAC
Broken Access Control
OWASP TOP 10 · A01
►
Definition
A security flaw permitting unauthorized users to access restricted resources, perform actions beyond their privilege level, or view another user's data due to improper enforcement of access policies.
Example
A web app exposes user records at /api/users/[id] without verifying the requesting user's ownership — allowing any authenticated user to view another user's private data by changing the ID.
BB
Bug Bounty
VDP · CROWDSOURCED SECURITY
►
Definition
A program offering financial rewards to external security researchers for responsibly discovering and reporting vulnerabilities in software or systems. Extends testing coverage beyond internal teams.
Example
A tech company launches a HackerOne bug bounty program offering $10,000–$100,000 for critical RCE vulnerabilities in their production platform, attracting global security researchers.
CC
Compensating Control
RISK MITIGATION
►
Definition
An alternative security measure implemented when a primary control cannot be deployed due to technical, operational, or financial constraints. Must provide equivalent risk reduction as the original control.
Example
A legacy medical device cannot be patched for a known CVE; the organization implements network segmentation and enhanced monitoring as compensating controls to reduce risk exposure.
CVE
Common Vulnerabilities & Exposures
CVE
►
Definition
A public dictionary of standardized identifiers for known security vulnerabilities in software and hardware. Maintained by MITRE; each CVE provides a unique ID, description, and references for a specific flaw.
Example
CVE-2021-44228 (Log4Shell) identifies a critical RCE vulnerability in Apache Log4j 2, allowing analysts and administrators to quickly locate patches and mitigation guidance.
CVSS
Common Vulnerability Scoring System
CVSS
►
Definition
An open framework for communicating the characteristics and severity of software vulnerabilities via a numerical score (0–10). Considers attack vector, complexity, privileges required, user interaction, and impact.
Example
A vulnerability with a CVSS score of 9.8 (Critical) is prioritized for immediate patching, while a score of 3.1 (Low) is scheduled for the next maintenance window.
PTCH
Patch Management
VULNERABILITY REMEDIATION
►
Definition
The systematic process of acquiring, testing, and deploying software updates (patches) to fix vulnerabilities and improve system functionality, within defined SLA windows based on severity.
Example
A SOC establishes a patch SLA: Critical (CVSS ≥9) patched within 24h, High within 7 days, Medium within 30 days. Automated tools deploy patches and report compliance.
TM
Threat Modeling
SDLC SECURITY · STRIDE
►
Definition
A structured approach to identifying, categorizing, and prioritizing potential threats to a system during design. Common frameworks include STRIDE (Spoofing, Tampering, Repudiation, Info Disclosure, DoS, EoP) and PASTA.
Example
A development team applies STRIDE threat modeling to a new API gateway, identifying that the authentication endpoint is susceptible to spoofing and adding token validation as a mitigation.
