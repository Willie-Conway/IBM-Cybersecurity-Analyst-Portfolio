1.1 — Security Control Types
▶
TERMDEFINITIONCONTEXTUAL EXAMPLE
CategoriesBroad classifications describing the nature and purpose of security controls.Controls are often categorized as technical, managerial, operational, and physical to streamline implementation.
Definition
Broad classifications that describe the nature and purpose of security controls, enabling organized application across different domains of an organization.
Contextual Example
Security controls are categorized into technical, managerial, operational, and physical to streamline their implementation and management.
CompensatingAlternative controls that achieve security goals when primary controls are not feasible.When full encryption is not possible, strong access controls and logging serve as compensating controls.
Definition
Controls that provide an alternative method to achieve security goals when primary controls cannot be implemented due to technical, financial, or operational constraints.
Contextual Example
If an organization cannot implement a complete encryption solution, it might protect sensitive data with compensating controls such as strong access controls and logging.
Control TypesCategories of security mechanisms designed to mitigate risks, enforce policies, and safeguard systems.Preventive stops threats; detective identifies incidents; corrective restores systems; deterrent discourages attacks.
Definition
Categories of security mechanisms designed to mitigate risks, enforce policies, and safeguard systems. Includes preventive, detective, corrective, deterrent, compensating, and directive types.
Contextual Example
Organizations use preventive controls to stop threats before they occur, detective controls to identify incidents, and corrective controls to restore systems after a breach.
CorrectiveControls that respond to and rectify security incidents after they have occurred.Post-breach patching of vulnerabilities and updating security protocols are corrective controls.
Definition
Controls activated after an incident to minimize damage, restore operations, and prevent recurrence. Include patch management, system restoration, and policy revisions.
Contextual Example
After a data breach, an organization implements corrective controls such as patching vulnerabilities and updating security protocols to prevent future incidents.
DetectiveControls that identify and alert organizations to security incidents as they occur.Intrusion detection systems (IDS) monitor network traffic and alert administrators to suspicious activities.
Definition
Controls that monitor systems and activities to identify security incidents in progress or after the fact. They do not prevent attacks but provide visibility and trigger response.
Contextual Example
IDS monitors network traffic and alerts administrators to suspicious activities, serving as detective controls.
DeterrentControls designed to discourage individuals from attempting to breach security measures.Warning signs about surveillance cameras deter potential intruders from attempting unauthorized access.
Definition
Controls that reduce the likelihood of an attack by discouraging adversaries through visible warnings, legal notices, or security presence. Target attacker psychology.
Contextual Example
Warning signs indicating surveillance cameras are in use act as deterrent controls, making potential intruders think twice before attempting to breach security.
DirectiveControls that guide employee behavior and establish expectations for organizational security practices.A security awareness training program teaching employees to recognize phishing is a directive control.
Definition
Controls that guide behavior through policies, procedures, standards, and training. Establish expectations and required actions rather than technical enforcement.
Contextual Example
A security awareness training program that educates employees on phishing attacks serves as a directive control, guiding them on recognizing and responding to security threats.
ManagerialControls focusing on managing security policies, procedures, and compliance through organizational practices.Security policies and employee training programs are managerial controls ensuring security protocols are followed.
Definition
Controls addressing the human and policy dimension of security. Include risk assessments, security policies, awareness training, and compliance audits. Also called administrative controls.
Contextual Example
A company's security policy and employee training programs are managerial controls that ensure security protocols are followed.
OperationalControls involving day-to-day operations and processes to ensure security is maintained.Regular security audits and incident response drills are operational controls supporting effective threat response.
Definition
Controls implemented and executed by people as part of ongoing operations. Include security monitoring, patch cycles, change management, and incident response procedures.
Contextual Example
Regular security audits and incident response drills are operational controls that help an organization respond effectively to security threats.
PhysicalSecurity measures designed to protect physical assets and facilities from unauthorized access or damage.Key card access systems and biometric scanners restricting server room entry are physical controls.
Definition
Controls protecting tangible assets, facilities, and personnel from unauthorized physical access, theft, damage, or environmental hazards. Operate independently of digital systems.
Contextual Example
Access control systems such as key cards or biometric scanners are physical controls that restrict entry to sensitive areas.
PreventiveControls aimed at stopping security incidents before they occur.Antivirus software and regular patch management are preventive controls protecting systems from malware.
Definition
Controls that proactively reduce the likelihood or impact of a security incident. Include firewalls, encryption, MFA, access controls, and security hardening.
Contextual Example
Installing antivirus software and performing regular updates are preventive controls that help protect systems from malware attacks.
TechnicalSecurity measures using technology and software to protect information systems.Firewalls and TLS encryption are technical controls safeguarding data transmitted over networks.
Definition
Controls (also called logical controls) using technology to enforce security policies automatically. Include firewalls, IDS/IPS, encryption, ACLs, and authentication systems.
Contextual Example
Firewalls and encryption are examples of technical controls used to safeguard data during transmission over networks.
1.2 — Fundamental Security Concepts
▶
TERMDEFINITIONCONTEXTUAL EXAMPLE
Access BadgeA physical card or token authorized personnel use to gain access to restricted areas.Employees scan badges at entry points to unlock doors to secure areas within an office.
Definition
A physical or smart card containing encoded credentials (magnetic strip, RFID, or smart chip) that authenticates the holder and grants access to controlled physical spaces.
Contextual Example
Employees must scan their access badges at entry points to unlock doors and access secure areas within an office.
Access Control VestibuleAn enclosed area restricting building access, requiring identity verification before entry. Also called a mantrap.Corporate offices use vestibules with security guards to verify identities before allowing entry into secured areas.
Definition
An interlocking door system preventing tailgating. The first door must close and authentication must complete before the second door opens. Also called a mantrap.
Contextual Example
Many corporate offices use access control vestibules with security guards to verify identities before allowing entry.
Adaptive IdentityThe ability to adjust authentication methods based on the context and risk associated with access requests.An organization requires MFA for high-risk or unusual transactions, adapting to potential threats dynamically.
Definition
Systems that evaluate contextual signals — location, device posture, time, behavior — and dynamically adjust authentication requirements. Higher risk triggers stronger challenges.
Contextual Example
An organization can require multi-factor authentication for high-risk transactions, adapting to potential threats.
Authenticating PeopleThe process of verifying the identity of a user attempting to access a system or service.Requiring a username and password is a common method for authenticating users before granting access.
Definition
The process of verifying a human user's identity through credentials they know (password), possess (token), or are (biometric) before granting system access.
Contextual Example
Requiring a username and password is a common method for authenticating users before granting access to a system.
Authenticating SystemsThe process of verifying the identity of devices or systems within a network to ensure secure communication.Mutual TLS (mTLS) authenticates both the client and server in a secure connection.
Definition
Verifying the identity of machines, devices, or services — not human users — using certificates, shared secrets, or hardware identifiers before allowing communication.
Contextual Example
Mutual TLS (Transport Layer Security) authenticates both the client and server in a secure connection, ensuring both parties are legitimate.
Authentication, Authorization & Accounting (AAA)A framework for controlling access by verifying identities, granting permissions, and tracking usage.A bank uses AAA protocols to ensure only authorized users access accounts and all transactions are logged.
Definition
AAA is the foundational access control framework: Authentication verifies who you are; Authorization determines what you can do; Accounting records what you did. RADIUS and TACACS+ are common AAA protocols.
Contextual Example
A bank uses AAA protocols to ensure that only authorized users can access their accounts and that their transactions are logged for audit purposes.
Authorization ModelsFrameworks determining how access rights are granted to users and systems.Role-Based Access Control (RBAC) grants permissions based on user roles, simplifying management.
Definition
Models include DAC (owner controls access), MAC (mandatory labels/clearances), RBAC (role determines access), ABAC (attribute-based dynamic policies), and Rule-Based (fixed rules for all).
Contextual Example
Role-Based Access Control (RBAC) grants permissions based on user roles within an organization, simplifying management.
AvailabilityThe guarantee that systems and data are accessible when needed by authorized users. Part of the CIA triad.Redundant servers and off-site backups ensure high availability of services during outages.
Definition
The 'A' in the CIA triad. Ensures authorized users can access systems and data when needed. Threatened by DoS attacks, hardware failures, and natural disasters.
Contextual Example
Implementing redundant servers and backups ensures high availability of services during outages.
BollardsShort, sturdy vertical posts that control vehicle access and protect infrastructure from ramming attacks.Bollards in front of government buildings prevent unauthorized vehicle entry and protect against vehicle-borne threats.
Definition
Vehicle access control devices — fixed or removable steel or concrete posts — that prevent unauthorized vehicles from entering a protected area. Protect against vehicle-borne IED attacks.
Contextual Example
Bollards are often used in front of government buildings to prevent unauthorized vehicle entry and enhance security.
ConfidentialityThe principle ensuring sensitive information is not disclosed to unauthorized individuals. Part of the CIA triad.Encrypting emails containing personal information maintains confidentiality during transmission.
Definition
The 'C' in the CIA triad. Ensures data is accessible only to those with authorized access. Protected through encryption, access controls, data classification, and need-to-know policies.
Contextual Example
Encrypting emails to protect personal information is a common practice to maintain confidentiality.
Control PlaneThe network/system component responsible for managing traffic, implementing policies, and making routing decisions.In zero-trust architecture, the control plane enforces that only authenticated users access sensitive resources.
Definition
The management layer of a network that makes decisions about where traffic should go. In SDN and zero-trust architectures, it enforces policy by dynamically managing the data plane's behavior.
Contextual Example
In a zero-trust architecture, the control plane ensures that only authenticated and authorized users can access sensitive resources by dynamically applying security policies.
Data PlaneThe network layer responsible for the actual transfer and processing of user data packets.The data plane handles packet flow while enforcing policies established by the control plane.
Definition
The forwarding plane that carries user traffic and applies access decisions made by the control plane. Operates at wire speed and is responsible for packet forwarding, filtering, and traffic shaping.
Contextual Example
The data plane handles the flow of data packets while enforcing policies established by the control plane.
Deception & Disruption TechnologySecurity technologies designed to mislead and disrupt attackers by creating fake resources and decoys.Honeypots lure cyber attackers away from real systems, letting security teams analyze their tactics.
Definition
Technologies creating fabricated environments, credentials, and decoy assets to detect attackers early, waste their time, gather intelligence, and mislead them away from real targets.
Contextual Example
Using a honeypot to lure cyber attackers away from real systems allows security teams to analyze and respond to threats.
FencingBarriers constructed to prevent unauthorized physical access to a facility or area.Chain-link fencing around a data center provides a physical barrier against unauthorized intruders.
Definition
A physical perimeter control using barriers to restrict unauthorized access to a facility. Often the first layer in a defense-in-depth physical security strategy.
Contextual Example
Chain-link fencing around a data center provides a physical barrier against unauthorized intruders attempting to access the facility.
Gap AnalysisComparing actual security performance with desired performance to identify discrepancies and improvement areas.A company evaluates its cybersecurity measures against industry best practices to identify shortfalls.
Definition
A process identifying the difference between an organization's current security posture and a target state. The output is a prioritized remediation roadmap aligned to a framework or regulation.
Contextual Example
A company conducts a gap analysis to evaluate its current cybersecurity measures against industry best practices.
HoneyfileA file planted in a system that appears valuable but is designed to alert administrators when accessed.A file labeled "Confidential Financial Data" triggers an alert if any user attempts to open it.
Definition
A decoy document placed where an attacker would likely access during reconnaissance. Any read, write, or access attempt triggers an alert — zero false-positives since no legitimate use exists.
Contextual Example
A company might place a honeyfile labeled "Confidential Financial Data" to trigger an alert if someone attempts to open it.
HoneynetA network of honeypots simulating a real environment to capture a broader range of attack behavior data.Security researchers deploy honeynets to study APTs by observing how attackers move across multiple decoy systems.
Definition
A controlled network of interconnected honeypots simulating an entire production environment, enabling researchers to observe complete attack kill-chains from initial access through lateral movement.
Contextual Example
Security researchers deploy a honeynet to study advanced persistent threats (APTs) by observing how attackers interact with multiple decoy systems.
HoneypotA decoy system designed to attract and detect unauthorized access attempts or malicious activity.A honeypot mimics a vulnerable server to monitor attacker behavior and gather threat intelligence.
Definition
A purposely vulnerable decoy system that appears to be a legitimate target. Detects unauthorized access, captures attacker tools and techniques, and generates early-warning alerts with zero false-positives.
Contextual Example
A honeypot can mimic a vulnerable server to monitor attacker behavior and gather intelligence on security threats.
HoneytokenA piece of data appearing legitimate that serves as a trap to detect unauthorized access or misuse.A fake user account (honeytoken) lets security teams monitor unauthorized attempts to use stolen credentials.
Definition
Digital decoys — fake credentials, API keys, database records, or email addresses — with no legitimate use. Any attempt to use them is an unambiguous indicator of compromise.
Contextual Example
Using a honeytoken, such as a fake user account, allows security teams to monitor unauthorized attempts to access sensitive resources.
Implicit Trust ZonesAreas within a network automatically trusted without verification, posing significant security risks.Legacy systems trusting all internal network traffic create implicit trust zones vulnerable to lateral movement.
Definition
Network areas or segments where traffic is trusted by default without individual authentication or authorization. Represent a significant risk in modern environments where zero trust is preferred.
Contextual Example
Legacy systems that trust all traffic within the corporate network can create implicit trust zones vulnerable to attacks.
InfraredA sensor type that detects heat emitted by objects, commonly used in security systems for motion detection.Infrared sensors in alarm systems trigger alerts when detecting unauthorized movement in restricted areas.
Definition
Passive infrared (PIR) sensors detect changes in heat radiation emitted by moving objects. Widely used in motion-activated alarms and lighting systems for physical security.
Contextual Example
Infrared sensors are commonly used in alarm systems to trigger alerts when detecting unauthorized movement.
IntegrityThe assurance that data is accurate and unaltered except by authorized users. Part of the CIA triad.Hashing files ensures data integrity, allowing users to verify a file has not been modified since creation.
Definition
The 'I' in the CIA triad. Ensures data has not been tampered with by unauthorized parties. Protected through cryptographic hashing, digital signatures, checksums, and version control.
Contextual Example
Hashing files can ensure data integrity, allowing users to verify that the file has not been modified since its creation.
LightingThe strategic use of light enhances security by illuminating areas and deterring criminal activity.Motion-activated lighting around a building's perimeter discourages unauthorized access at night.
Definition
A physical security control that deters unauthorized access and aids detection. Both ambient and motion-triggered lighting eliminate blind spots and reduce attacker concealment opportunities.
Contextual Example
Motion-activated lighting around a building's perimeter increases visibility and discourages unauthorized access at night.
MicrowaveSensors using microwave signals to detect motion, commonly used in outdoor security systems.Microwave sensors detect movement across large outdoor areas where infrared sensors may be impractical.
Definition
Active sensors emitting microwave signals and detecting changes in the reflected signal caused by moving objects. Used for perimeter protection, especially outdoors where environmental factors affect infrared sensors.
Contextual Example
Microwave sensors can be used in outdoor security systems to detect movement across large areas.
Non-RepudiationA security principle preventing individuals from denying their actions related to data or transactions.Digital signatures provide non-repudiation, proving the sender sent a specific message or transaction.
Definition
Ensures the origin of data or a transaction cannot be denied. Achieved through digital signatures, audit logs, and timestamping. Critical for legal admissibility and financial transactions.
Contextual Example
Digital signatures provide non-repudiation, allowing the sender to prove that they sent a message or transaction.
Physical SecurityMeasures designed to protect physical assets from unauthorized access or damage.Security cameras and trained security personnel are components of a comprehensive physical security strategy.
Definition
The protection of hardware, software, networks, and data from physical actions and events that could cause serious loss or damage to an enterprise or organization.
Contextual Example
Installing security cameras and hiring security personnel are components of a comprehensive physical security strategy.
Policy AdministratorThe individual or system responsible for managing security policies and access controls.A cybersecurity manager acts as policy administrator, ensuring access controls align with organizational goals.
Definition
In zero-trust architectures, the policy administrator is the component or person that establishes and manages policies that govern access decisions, working with the policy engine and enforcement points.
Contextual Example
A cybersecurity manager is the policy administrator, ensuring access controls align with organizational security goals.
Policy Enforcement PointThe point within a network where access control policies are applied and enforced.A firewall acts as a policy enforcement point, blocking unauthorized access based on established security rules.
Definition
In zero-trust architecture, the PEP is the component that intercepts all resource requests and enforces the access decisions made by the policy engine. Firewalls and proxies serve this role.
Contextual Example
A firewall acts as a policy enforcement point, blocking unauthorized access based on established security rules.
Policy EngineThe component that evaluates access requests against defined security policies to determine access rights.In zero-trust architecture, the policy engine assesses requests based on user identity and context before granting access.
Definition
The decision-making core of a zero-trust system. Evaluates access requests against policy, considering identity, device health, location, and resource sensitivity to grant or deny access.
Contextual Example
In a zero-trust architecture, the policy engine assesses requests based on user identity and context before granting access.
Policy-Driven Access ControlAccess control mechanisms governed by established security policies that automate access decisions.Predefined policies automatically grant or deny resource access based on user roles and contextual attributes.
Definition
Uses defined rules and automated decision engines to evaluate access requests against organizational policy, eliminating manual decisions and ensuring consistent, scalable enforcement.
Contextual Example
Using predefined security policies, an organization can automatically grant or deny access to resources based on user roles.
PressureSensors that detect changes in pressure, often used to detect unauthorized access or breaches.Pressure mats placed in secure areas alert security personnel if someone steps on them.
Definition
Pressure-sensitive sensors or mats detect the weight of a person or object. Used in vaults, secure rooms, or restricted pathways to alert when someone is physically present.
Contextual Example
Pressure mats can be placed in secure areas to alert security personnel if someone steps on them.
Security GuardA trained individual who monitors and protects property and individuals from unauthorized access or threats.Security guards stationed at entrances monitor access and respond to security incidents in corporate environments.
Definition
A human physical security control providing deterrence, detection, and response capabilities. Guards can make contextual judgments that automated systems cannot, especially for social engineering scenarios.
Contextual Example
Security guards are stationed at entrances to monitor access and respond to security incidents in corporate environments.
SensorsDevices that detect specific physical changes or intrusions in a secure area.Infrared sensors trigger alarms when motion is detected in restricted areas after hours.
Definition
Physical detection devices — including infrared, microwave, ultrasonic, and pressure sensors — that monitor the physical environment and trigger alerts when unauthorized activity is detected.
Contextual Example
Infrared sensors can trigger alarms if motion is detected in restricted areas after hours.
Subject / SystemEntities in a security policy: a subject requests access; a system is the resource being accessed.In access control, the subject (user) requests access to a system (database) based on defined policies.
Definition
In access control models, the subject is any user or process requesting access to a resource. The system (object) is the resource. Access decisions evaluate whether the subject has rights to the system.
Contextual Example
In access control, the subject (a user) requests access to a system (a database) based on defined policies.
Threat Scope ReductionMeasures taken to limit the potential impact of security threats on an organization.Network segmentation isolating sensitive data reduces the blast radius in the event of a breach.
Definition
Architectural and operational controls minimizing how far a threat can propagate. Techniques include network segmentation, microsegmentation, least-privilege access, and asset isolation.
Contextual Example
Segmenting networks to isolate sensitive data reduces the threat scope in case of a breach.
UltrasonicSensors using sound waves to detect motion and environmental changes, commonly used in security systems.Ultrasonic sensors create invisible barriers that trigger alarms if an intruder enters the protected zone.
Definition
Active sensors emitting high-frequency sound waves and measuring reflections. Detect motion and presence by analyzing changes in reflected sound patterns. Used in indoor and vault security.
Contextual Example
Ultrasonic sensors can be employed in security systems to create invisible barriers that trigger alarms if breached.
Video SurveillanceVideo cameras that monitor and record activity in and around a facility for security purposes.Retail stores use video surveillance systems to deter theft and provide evidence of security incidents.
Definition
CCTV and IP camera systems providing continuous monitoring, recording, and deterrence. Modern systems include analytics for automated motion detection, facial recognition, and anomaly alerts.
Contextual Example
Retail stores utilize video surveillance systems to deter theft and monitor customer behavior.
Zero TrustA security model requiring strict identity verification for every access request, regardless of network location.In zero-trust environments, all access requests are authenticated and authorized even for internal users.
Definition
Operates on "never trust, always verify." Every access request — inside or outside the perimeter — must be authenticated, authorized, and continuously validated against policy. Eliminates implicit trust zones.
Contextual Example
In a zero-trust environment, all access requests must be authenticated and authorized, even for users within the corporate network.
1.3 — Change Management & Security Impact
▶
TERMDEFINITIONCONTEXTUAL EXAMPLE
Allow Lists / Deny ListsSecurity mechanisms specifying which entities are permitted or blocked from accessing resources.An allow list permits only trusted IP addresses to access internal applications, blocking everything else.
Definition
Allow lists (whitelists) explicitly permit only approved entities; everything else is denied by default. Deny lists (blacklists) block known-bad entities. Allow lists provide stronger security but require more management.
Contextual Example
An organization uses an allow list to permit only trusted IP addresses to access its internal applications, enhancing security.
Application RestartThe action of closing and reopening an application to implement updates or fixes.An application restart is required after applying security patches to ensure vulnerabilities are fully addressed.
Definition
A change management action requiring applications to be stopped and restarted to apply configuration changes, patches, or updates. Must be coordinated with users to minimize disruption.
Contextual Example
Restarting the application is required after applying security patches to ensure vulnerabilities are addressed.
Approval ProcessA defined workflow requiring authorization before proceeding with changes or actions within an organization.New security protocols must be reviewed and approved by IT and compliance teams before implementation.
Definition
A change management workflow ensuring proposed changes are reviewed for security impact, technical feasibility, and business risk before implementation. Typically managed by a Change Advisory Board (CAB).
Contextual Example
New security protocols must go through an approval process that includes reviews by IT and compliance teams.
Backout PlanA predefined strategy for reversing changes if an implementation fails or causes adverse effects.A backout plan restored previous configurations after a new security software deployment caused system instability.
Definition
A rollback plan specifying exactly how to restore prior state if a change causes unintended problems. Required component of any change request in a mature change management process.
Contextual Example
A backout plan was activated to restore previous configurations after the deployment of a new security software caused system instability.
Business Processes Impacting SecurityOrganizational procedures and workflows that can affect security measures and incident response.An approval process for software purchases ensures the organization only uses compliant, vetted tools.
Definition
Business workflows — procurement, onboarding, offboarding, software deployment — that directly impact security operations. Understanding these intersections prevents gaps in security coverage.
Contextual Example
An approval process for new software purchases can impact security operations by ensuring the organization only uses compliant tools.
DependenciesRelationships between systems or components where one relies on another for functionality or data access.A new security feature may depend on existing systems, requiring careful coordination during implementation.
Definition
In change management, understanding upstream and downstream dependencies is critical. Modifying one component may break dependent systems, potentially creating security gaps or outages.
Contextual Example
A new security feature might have dependencies on existing systems, requiring careful coordination during implementation.
DocumentationCreating and maintaining records describing systems, policies, and procedures for reference and compliance.Updating documentation after a system upgrade ensures all users have access to current procedures.
Definition
Security documentation — policies, procedures, diagrams, runbooks, baselines — supports incident response, audit readiness, and onboarding. Must be updated after every significant change.
Contextual Example
Updating documentation after a system upgrade ensures that all users have access to current procedures and security protocols.
DowntimePeriods when a system or service is unavailable due to maintenance, updates, or unexpected failures.Scheduled downtime for system updates is communicated in advance to minimize business disruption.
Definition
A direct threat to the availability pillar of the CIA triad. Planned downtime (maintenance windows) must be communicated; unplanned downtime requires immediate incident response.
Contextual Example
Scheduled downtime for system updates is communicated to users in advance to minimize disruption to business operations.
Impact AnalysisEvaluating the potential effects of changes to security operations on business functions.Conducting an impact analysis before implementing new security measures helps ensure minimal disruption to critical operations.
Definition
A systematic assessment of how a proposed change could affect business operations, security posture, and compliance. Mandatory before implementing changes that could disrupt critical services.
Contextual Example
Conducting an impact analysis before implementing new security measures helps ensure minimal disruption to critical business operations.
Legacy ApplicationsOutdated software still in use but not fully supported or compliant with current security standards.Legacy applications often require additional security controls to compensate for their outdated technology.
Definition
Outdated software systems still in use but no longer receiving vendor support or security updates. Represent significant risk due to unpatched vulnerabilities and inability to meet modern security requirements.
Contextual Example
Legacy applications often require additional security measures to protect against vulnerabilities due to their outdated technology.
Maintenance WindowA scheduled time period allocated for performing updates or repairs to systems without impacting normal operations.An organization scheduled a weekend maintenance window to apply critical security patches to all servers.
Definition
A pre-approved time period — often nights or weekends — during which changes can be applied to systems with minimal business impact. Part of formal change management processes.
Contextual Example
The organization scheduled a maintenance window over the weekend to apply critical security patches to all servers.
OwnershipThe designation of responsibility for specific assets, processes, or data within an organization.Data ownership defines who is accountable for maintaining the security and integrity of sensitive customer information.
Definition
In change management and data governance, ownership assigns clear accountability for assets. Owners are responsible for authorizing changes, ensuring compliance, and maintaining security of their assets.
Contextual Example
Data ownership defines who is accountable for maintaining the security and integrity of sensitive customer information.
Restricted ActivitiesActions limited or prohibited to reduce risk and enhance security posture within an organization.Remote access to sensitive systems can be a restricted activity to minimize exposure to potential threats.
Definition
Specific actions defined in policy as prohibited or requiring special authorization. Commonly includes copying data to removable media, remote access to production systems, and shadow IT adoption.
Contextual Example
Remote access to sensitive systems can be a restricted activity to minimize exposure to potential threats.
Service RestartThe process of stopping and starting a service to apply changes or resolve issues.A service restart is performed after updating security software to ensure new configurations take effect.
Definition
A change management action required to activate configuration changes in running services. Unlike an application restart, a service restart may be performed without user interaction on server-side components.
Contextual Example
A service restart is performed after updating the security software to ensure the new configurations take effect.
StakeholdersIndividuals or groups with an interest in or affected by security operations and decisions.Stakeholders in a security project include IT staff, management, compliance officers, and end-users.
Definition
Anyone with a vested interest in the outcome of security decisions. Effective change management requires identifying and communicating with all relevant stakeholders to ensure buy-in and minimize disruption.
Contextual Example
Stakeholders in a security project can include IT staff, management, compliance officers, and end-users who rely on secure systems.
Standard Operating Procedure (SOP)Step-by-step instructions outlining how to perform specific tasks to ensure consistency and quality in operations.An SOP for incident response provides detailed steps for detecting, reporting, and addressing security incidents.
Definition
Documented, repeatable procedures ensuring tasks are performed consistently and correctly regardless of who executes them. SOPs are essential for incident response, change management, and audit compliance.
Contextual Example
An SOP for incident response provides detailed steps for detecting, reporting, and addressing security incidents systematically.
Technical ImplicationsThe consequences or effects that changes in technology or systems can have on security operations and policies.Migration to a cloud-based service has technical implications including changes to data encryption and access controls.
Definition
The downstream technical effects of implementing a change, which must be assessed during impact analysis. Include effects on performance, compatibility, security controls, and compliance posture.
Contextual Example
The migration to a cloud-based service has technical implications, including changes to data encryption and access controls.
Test ResultsOutcomes of security testing — assessments, audits, and penetration tests — providing insights into vulnerabilities.Test results from a recent security audit revealed critical vulnerabilities in the network configuration.
Definition
Documented findings from security testing activities. Used to drive remediation priorities, demonstrate compliance, and verify that controls are working as intended after changes are applied.
Contextual Example
Test results from a recent security audit revealed critical vulnerabilities in the network configuration that need immediate attention.
Updating DiagramsRevising visual representations of systems and processes to reflect current configurations or workflows.Network diagrams are updated to include new security appliances and reflect system architecture changes.
Definition
Maintaining accurate network and system diagrams is critical for incident response, security assessments, and change management. Outdated diagrams can lead to misconfigured security controls.
Contextual Example
Network diagrams are updated to include new security appliances and reflect system architecture changes.
Updating Policies / ProceduresRevising organizational rules and guidelines to address new risks, technologies, or regulatory requirements.Updating security policies after a data breach ensures lessons learned are integrated into future practices.
Definition
A continuous process ensuring security policies remain relevant and effective. Triggered by incidents, technology changes, regulatory updates, or audit findings. Requires review, approval, and communication to stakeholders.
Contextual Example
Updating security policies after a data breach ensures that lessons learned are integrated into future practices.
Version ControlA system for managing changes to documents, software, or other information, tracking revisions and enabling collaboration.Version control helps teams track changes to security documentation, ensuring everyone uses the most current information.
Definition
Systems like Git track changes to code, configuration files, and documentation over time. Enables rollback to known-good states, provides audit trails, and prevents unauthorized or accidental changes.
Contextual Example
Version control systems help teams track changes to security documentation, ensuring all team members are working with the most current information.
1.4 — Cryptographic Solutions
▶
TERMDEFINITIONCONTEXTUAL EXAMPLE
Asymmetric EncryptionAn encryption method using a pair of keys (public and private) for secure communication.Asymmetric encryption allows secure data exchange without sharing private keys, enhancing security.
Definition
Uses mathematically linked key pairs: data encrypted with the public key can only be decrypted by the corresponding private key. Slower than symmetric encryption; used for key exchange and digital signatures.
Contextual Example
Asymmetric encryption allows for the exchange of secure data without sharing private keys, enhancing security.
BlockchainA decentralized digital ledger that securely records transactions across multiple computers.Blockchain underpins cryptocurrencies like Bitcoin, providing a tamper-evident method for recording transactions.
Definition
An immutable, distributed ledger where each block contains a cryptographic hash of the previous block, creating a chain. Tampering with any block invalidates all subsequent blocks.
Contextual Example
Blockchain technology underpins cryptocurrencies like Bitcoin, providing a secure method for recording transactions.
Certificate AuthoritiesTrusted entities that issue digital certificates and verify the identities of organizations or individuals.Certificate authorities provide SSL/TLS certificates to websites, enabling secure encrypted connections.
Definition
Trusted third parties in the PKI hierarchy that issue, sign, and revoke digital certificates. They validate identity before issuing certificates, establishing chain-of-trust in cryptographic systems.
Contextual Example
Certificate authorities, such as Let's Encrypt, provide SSL certificates to websites, enabling secure connections.
Certificate Revocation Lists (CRLs)Lists maintained by certificate authorities identifying revoked certificates no longer valid.Web browsers check CRLs to verify an SSL certificate has not been revoked before establishing a secure connection.
Definition
A signed list published by a CA containing serial numbers of certificates that have been revoked before their expiration date. Clients check CRLs (or OCSP) to avoid trusting compromised certificates.
Contextual Example
CRLs are used by web browsers to check if an SSL certificate has been revoked before establishing a secure connection.
Certificate Signing Request (CSR)The process of creating a request for a digital certificate that includes the public key and identity information.Organizations generate a CSR to obtain an SSL certificate from a CA, ensuring secure website connections.
Definition
A formatted message submitted to a CA containing the applicant's public key and identifying information. The CA verifies the information and, if approved, signs the certificate with its private key.

Organizations generate a CSR to obtain an SSL certificate from a certificate authority, ensuring secure website connections.
CertificatesDigital documents that verify the identity of entities and bind them to cryptographic keys.SSL/TLS certificates secure website communications, ensuring users are connected to legitimate services.
Definition
X.509 digital certificates bind a public key to an identity, signed by a trusted CA. Contain the subject's name, public key, validity period, issuing CA, and digital signature.
Contextual Example
SSL certificates secure website communications, ensuring that users are connected to legitimate services.

Data masking is often used in non-production environments to protect sensitive customer data during testing.

Protects data at rest within a database. Can be transparent (TDE — entire database encrypted) or column-level (specific sensitive fields encrypted). Protects against storage media theft and unauthorized access.

Implementing database encryption helps secure customer information stored in an e-commerce platform.

Created by hashing a message and encrypting the hash with the sender's private key. Recipients decrypt with the sender's public key and verify the hash matches. Provides authentication, integrity, and non-repudiation.

Digital signatures ensure that an email is from the claimed sender and has not been altered during transmission.
EncryptionThe process of converting plaintext into ciphertext to protect data from unauthorized access.Encrypting credit card information ensures it remains secure during transmission across networks.
Definition
A cryptographic process transforming readable data (plaintext) into an unreadable format (ciphertext) using an algorithm and key. Only parties with the correct decryption key can recover the original data.
Contextual Example
Encrypting sensitive data such as credit card information helps ensure it is secure during transmission.
File EncryptionEncrypting individual files to protect their contents from unauthorized access.Encrypting a financial report file ensures only authorized users can view its contents.
Definition
Applies encryption at the individual file level, protecting specific files regardless of where they are stored or transmitted. More granular than full-disk encryption; allows selective protection.
Contextual Example
Encrypting a financial report file ensures that only authorized users can view its contents.
Full-Disk EncryptionEncrypting all data on a disk drive to protect information from unauthorized access.Full-disk encryption on laptops safeguards all data if the device is lost or stolen.
Definition
Encrypts the entire contents of a storage device, including the OS, applications, and user data. Data is decrypted on-the-fly during normal use; protects against physical device theft.
Contextual Example
Full-disk encryption is commonly used on laptops to safeguard data if the device is lost or stolen.
Hardware Security Module (HSM)A dedicated hardware device that manages and protects cryptographic keys in a secure environment.Financial institutions use HSMs to safeguard sensitive transaction data and cryptographic keys.
Definition
A tamper-resistant hardware device providing secure key generation, storage, and cryptographic operations. Keys never leave the HSM in plaintext. Used for PKI roots, code signing, and payment processing.
Contextual Example
Financial institutions often use HSMs to safeguard sensitive transaction data and cryptographic keys.
HashingConverting data into a fixed-size string (hash value) unique to the original data. A one-way process.Hashing passwords ensures the actual passwords remain unrecoverable even if the database is compromised.
Definition
A one-way mathematical function producing a fixed-length digest of any input. Used for password storage, integrity verification, and digital signatures. Cannot be reversed to obtain the original data.
Contextual Example
Hashing passwords ensures that even if the database is compromised, the actual passwords remain secure and unrecoverable.
Key EscrowStoring a copy of a private key with a trusted third party to allow recovery if the original is lost.Organizations use key escrow services to ensure encrypted data can be accessed in emergencies.
Definition
A key management practice where a copy of cryptographic keys is held by a trusted third party. Allows lawful access to encrypted data by authorities or recovery if keys are lost.
Contextual Example
Organizations often use key escrow services to ensure that encrypted data can be accessed in emergencies.
Key ExchangeThe process of securely sharing cryptographic keys between parties to enable encrypted communication.The Diffie-Hellman protocol allows two parties to generate a shared key securely over an insecure channel.
Definition
A cryptographic protocol allowing two parties to establish a shared secret over an insecure channel without prior communication. Diffie-Hellman and ECDH are foundational key exchange algorithms.
Contextual Example
The Diffie-Hellman method is a popular key exchange protocol that allows two parties to generate a shared key securely.
Key LengthThe size of the cryptographic key, which influences the strength and security of the encryption.Longer keys such as 256-bit are typically more secure and harder to brute-force than shorter keys.
Definition
The number of bits in a cryptographic key. Longer keys exponentially increase the number of possible combinations, making brute-force attacks impractical. AES-256 is the current standard for symmetric encryption.
Contextual Example
Longer key lengths, such as 256 bits, are typically more secure and harder to crack than shorter keys.
Key Management SystemA system managing the generation, distribution, storage, and revocation of cryptographic keys.A KMS ensures proper key lifecycle management, maintaining security across an organization's encrypted data.
Definition
Software or hardware systems automating the full lifecycle of cryptographic keys: generation, distribution, rotation, storage, backup, and revocation. Centralizes key management to reduce risk of exposure.
Contextual Example
A key management system helps organizations maintain security by ensuring proper key lifecycle management.
Key StretchingA technique increasing the security of a key by applying a cryptographic function multiple times.Key stretching makes brute-force attacks harder by increasing the computational time needed to test each key.
Definition
Takes a weak key (e.g., a user password) and applies hash functions repeatedly (PBKDF2, bcrypt, Argon2) to produce a stronger derived key. The added computation dramatically slows brute-force attacks.
Contextual Example
Key stretching makes brute-force attacks more difficult by increasing the time required to test each key candidate.
ObfuscationA technique making data or code unintelligible without proper authorization, without full encryption.Obfuscation techniques alter code or data formats to prevent unauthorized access while preserving functionality.
Definition
Making data or code difficult to understand without fully encrypting it. Used in software to prevent reverse engineering, and in data protection to obscure sensitive values without breaking functionality.
Contextual Example
Obfuscation techniques might involve altering code or data formats to prevent unauthorized access while preserving functionality.
Online Certificate Status Protocol (OCSP)A protocol used to obtain the revocation status of a digital certificate in real time.OCSP allows browsers to verify certificate validity quickly, enhancing security during online transactions.
Definition
A more efficient alternative to CRLs for checking certificate revocation. Clients query the OCSP responder in real time to get the current status of a certificate. OCSP Stapling improves performance.
Contextual Example
OCSP allows browsers to verify the validity of a certificate quickly, enhancing security during online transactions.
Open Public LedgerA transparent, accessible record of transactions that anyone can view, typically associated with blockchain.Open public ledgers enable users to verify transactions without relying on a central authority.
Definition
A distributed, publicly readable record of all transactions on a blockchain network. Transparency enables verification by any participant without requiring trust in a central authority.
Contextual Example
Open public ledgers enable users to verify transactions without relying on a central authority, enhancing trust.
Partition EncryptionEncrypting specific partitions on a storage device to protect sensitive data stored within those areas.Encrypting a partition containing confidential documents provides additional security beyond OS-level access controls.
Definition
Encrypts specific logical divisions of a storage device rather than the entire disk. Allows different encryption policies for different partitions — e.g., stronger encryption for a data partition vs. OS partition.
Contextual Example
Encrypting a partition that contains confidential business documents provides an additional layer of security.
Private KeyA secret cryptographic key used to decrypt data encrypted with the corresponding public key.A user keeps their private key secure, using it to decrypt messages sent with their public key.
Definition
In asymmetric cryptography, the private key is kept secret by the owner and used to decrypt incoming encrypted data and to create digital signatures. Its security is critical to the entire cryptographic system.
Contextual Example
A user keeps their private key secure, using it to decrypt messages received with their public key.
Public KeyA cryptographic key that can be shared publicly and used to encrypt data or verify digital signatures.In PKI, users share their public keys with others to enable secure encrypted communication.
Definition
In asymmetric cryptography, the public key is freely shareable and used by senders to encrypt data destined for the private key holder, or by recipients to verify digital signatures.
Contextual Example
In a PKI system, users share their public keys with others to enable secure communication.
Public Key Infrastructure (PKI)A framework managing digital certificates and public-key encryption to secure communications and transactions.PKI is used in secure email communication, ensuring only intended recipients can decrypt messages.
Definition
The complete system of hardware, software, policies, and procedures that manages digital certificates and public-private key pairs. Includes CAs, registration authorities, certificate stores, and revocation mechanisms.
Contextual Example
PKI is used in secure email communication, ensuring that only intended recipients can decrypt messages.
Record EncryptionEncrypting individual records within a database to enhance security at a granular level.Record encryption protects sensitive customer records while allowing other data to remain accessible.
Definition
The most granular form of database encryption — encrypting individual rows or records. Allows different encryption keys per record, providing fine-grained access control over sensitive data.
Contextual Example
Record encryption can protect sensitive customer records while allowing other data to remain accessible.
Root of TrustThe foundation of a security system establishing the legitimacy of entities and certificates within the system.The root of trust in PKI verifies the authenticity of certificates issued by trusted authorities.
Definition
An inherently trusted component — hardware (TPM, HSM) or software (root CA) — that forms the basis of a trust hierarchy. All security assertions in the system ultimately derive their trust from the root.
Contextual Example
The root of trust is crucial for PKI, as it verifies the authenticity of certificates issued by trusted authorities.
SaltingAdding a random value (salt) to input data before hashing to protect against precomputed attacks.Salting passwords before hashing prevents attackers from using rainbow tables to crack password hashes.
Definition
A random unique value appended to each password before hashing. Ensures that identical passwords produce different hashes, defeating rainbow table and dictionary attacks.
Contextual Example
Salting passwords before hashing prevents attackers from using rainbow tables to crack password hashes.
Secure EnclaveA secure area within a processor that protects sensitive data and operations from unauthorized access.Mobile devices use secure enclaves to protect biometric data like fingerprints and facial recognition.
Definition
A hardware-isolated execution environment within a CPU that processes sensitive operations with memory and execution isolated even from the main OS. Used for biometric authentication and key storage in modern devices.
Contextual Example
Secure enclaves, such as fingerprints and facial recognition, are utilized in mobile devices to protect biometric data.
Self-Signed CertificateA certificate signed by the entity that created it rather than a trusted certificate authority.Self-signed certificates are used for internal testing but are not trusted by browsers for public-facing sites.
Definition
A digital certificate signed with its own private key rather than a CA's. Provides encryption but not third-party identity validation. Browsers display warnings; suitable for internal systems and development only.
Contextual Example
Self-signed certificates are often used for internal testing but are not trusted by browsers for public-facing websites.
SteganographyThe process of hiding data within other non-secret data, making the hidden information difficult to detect.A secret message can be concealed within an image file without altering its visible appearance.
Definition
Conceals the existence of a message by embedding it within ordinary-looking files (images, audio, video). Unlike encryption (which hides content), steganography hides the fact that a message exists at all.
Contextual Example
Using steganography, a secret message can be concealed within an image file without altering its visible appearance.
Symmetric EncryptionAn encryption method using the same key for both encryption and decryption.Symmetric encryption is faster than asymmetric and used for encrypting large amounts of data like files or databases.
Definition
Uses a single shared key for both encryption and decryption. Much faster than asymmetric encryption. Key distribution is a challenge — the shared key must be securely exchanged before communication. AES is the standard.
Contextual Example
Symmetric encryption is faster and often used for encrypting large amounts of data, such as files or databases.
Third-Party CertificateCertificates issued by entities not directly involved in a transaction, playing a role in validation and trust.Third-party CAs validate organizational identities, providing trusted certificates for secure transactions.
Definition
Certificates issued by external, independent certificate authorities. Browsers and operating systems include trust stores pre-loaded with trusted third-party CA certificates, enabling automatic trust establishment.
Contextual Example
Third-party certificate authorities validate the identities of organizations, providing trusted certificates for secure transactions.
TokenizationReplacing sensitive data with non-sensitive equivalents (tokens) that can be used in its place.Tokenization replaces credit card numbers with unique tokens in payment processing, reducing PCI scope.
Definition
Substitutes sensitive data values with randomly generated tokens stored in a token vault. The original data is never transmitted or stored in business systems, dramatically reducing data breach impact.
Contextual Example
Tokenization is used in payment processing to replace credit card numbers with unique tokens, enhancing security.
Transport / Communication EncryptionEncrypting data transmitted over networks to protect it from interception during transit.HTTPS uses TLS transport encryption to secure web traffic, protecting user data during transmission.
Definition
Encryption applied to data in transit between systems. TLS/SSL is the primary protocol. Protects against eavesdropping and man-in-the-middle attacks on network communications.
Contextual Example
HTTPS uses transport encryption to secure web traffic, ensuring that user data is protected while being transmitted.
Trusted Platform Module (TPM)A hardware-based security device providing cryptographic functions and secure key storage.Computers use TPM to ensure secure boot and provide hardware-based encryption capabilities like BitLocker.
Definition
A dedicated microcontroller chip on the motherboard providing hardware-based security functions: secure key generation and storage, platform integrity measurement, and sealed storage tied to platform state.
Contextual Example
Computers often use TPM to ensure secure boot and hardware-based encryption capabilities.
Volume EncryptionEncrypting entire volumes (logical storage areas) on a disk to safeguard all data within.Volume encryption protects external hard drives containing sensitive information if the device is lost or stolen.
Definition
Encrypts a logical volume — a defined area of a disk that can span partitions or multiple drives. Provides comprehensive protection for all data within the volume, including temporary and swap files.
Contextual Example
Volume encryption is often used to protect external hard drives containing sensitive information.

Definition
Uses an asterisk (*) in the common name to secure all first-level subdomains of a domain. Reduces administrative overhead but if the private key is compromised, all covered subdomains are affected.

A company uses a wildcard SSL certificate to secure multiple subdomains with one certificate, reducing administrative overhead and cost.

Misinformation (unintentional) and disinformation (deliberate) attacks manipulate perception, damage reputations, undermine trust in institutions, or cause operational confusion. Used by nation-states and influence operations.

Attackers spread disinformation online to sow confusion during elections or disrupt organizational operations by creating fake news stories that mislead the public.

Network ports actively listening for connections that are not properly secured or should not be exposed. Attackers scan for open ports to discover running services, then target known vulnerabilities in those services.
Contextual Example
Leaving open service ports exposed without proper firewall rules increases the risk of external attacks such as unauthorized access or data breaches.

Definition
Mass social engineering attacks sent to broad audiences. Spear phishing targets specific individuals; whaling targets executives; vishing is via phone. The most common initial access vector in data breaches.
Contextual Example
A phishing email from a bank can deceive users into entering their login credentials on a fake website, leading to unauthorized access to their accounts.
PretextingA social engineering tactic where attackers create a fabricated scenario to extract sensitive information.An attacker pretends to be from tech support, convincing users to disclose their login credentials for "assistance."
Definition
Creating a false but plausible scenario (pretext) to manipulate targets into providing access or information. Requires research about the target to build credibility. Often used in vishing and insider attacks.
Contextual Example
In a pretexting scam, an attacker pretends to be from tech support, convincing users to disclose their login credentials under the guise of assisting with a problem.
Removable DevicePhysical devices like USB drives or external hard drives can carry malware and infect systems when connected.A USB stick labeled "Confidential Report" found in a parking lot installs malware when plugged in.
Definition
Physical media used to introduce malware into air-gapped or otherwise secure systems. The "USB drop" attack exploits human curiosity. Autorun features can execute malware immediately upon connection.
Contextual Example
A USB stick labeled "Confidential Report" is left in a public area. When an unsuspecting user plugs it in, it installs malware that compromises system security.
SmishingPhishing attacks conducted through SMS text messages, involving fraudulent links or urgent personal information requests.A smishing message from a fake bank urgently requests account verification via a link to a fraudulent website.
Definition
SMS-based phishing (smishing) exploits the higher open rates and lower suspicion associated with text messages. Often impersonates financial institutions, delivery services, or government agencies to create urgency.
Contextual Example
A smishing message appears to come from a financial institution, urgently requesting sensitive data via text to verify account information, leading users to a fraudulent website.
SMS-Based Attack VectorText messaging services on mobile phones, often targeted by attackers through smishing and social engineering.An SMS appearing to be from a bank asks recipients to confirm transactions by clicking a link to a fraudulent site.
Definition
The broader category of attacks leveraging SMS channels, including smishing, OTP interception attacks (SIM swapping), and malicious links embedded in promotional messages.
Contextual Example
An SMS that appears to be from a bank asks the recipient to confirm recent transactions by clicking a link, which redirects them to a fraudulent website collecting personal information.
Supply Chain Attack VectorThe network of vendors, suppliers, and service providers can introduce security risks if any component is compromised.A breach at an MSP exposes its client companies as attackers gain access through a trusted third party.
Definition
Attacks targeting less-secure elements in a supply chain — software vendors, hardware manufacturers, service providers — to compromise the ultimate target through trusted relationships. SolarWinds is the prime example.
Contextual Example
A breach at a managed service provider (MSP) can expose its client companies to potential security threats as attackers gain access through a trusted third party.
TyposquattingA cyberattack where attackers register domain names similar to legitimate ones, relying on common typing errors.An attacker registers "goolge.com" hoping users mistype "google.com" and visit the malicious fake site.
Definition
Also called URL hijacking. Registers misspelled or visually similar domain names to intercept users who make typing errors. The fake site may steal credentials, serve malware, or conduct phishing attacks.
Contextual Example
An attacker uses typosquatting by registering a domain like "goolge.com", hoping users mistype "google.com" and visit the fake site, which can steal their login information.
Unsecure NetworksNetworks without sufficient security measures, exposing them to eavesdropping and man-in-the-middle attacks.An attacker intercepts data on an unsecured public Wi-Fi network, capturing passwords and personal messages.
Definition
Networks lacking proper encryption, authentication, or access controls. Public Wi-Fi is a prime example. Attackers can eavesdrop on unencrypted traffic or deploy rogue access points for man-in-the-middle attacks.
Contextual Example
An attacker intercepts data transmitted over an unsecured public Wi-Fi network, capturing sensitive information such as passwords and personal messages from unsuspecting users.
Unsupported SystemsLegacy systems and applications no longer receiving security updates, making them highly vulnerable to attacks.Windows XP systems no longer receiving updates are prime targets for attackers exploiting unpatched vulnerabilities.
Definition
End-of-life systems that no longer receive vendor security patches. Vulnerabilities discovered after EOL remain permanently unpatched, making them valuable and persistent targets for attackers.
Contextual Example
Unsupported systems like Windows XP, which no longer receive updates, are prime targets for attackers looking to exploit unpatched vulnerabilities for unauthorized access.
Vendor Attack VectorSuppliers of products or services can introduce risks if their systems are compromised or poorly managed.An attacker compromises a software update vendor, inserting malware into updates affecting all clients.
Definition
Third-party vendors with access to organizational systems or data represent trusted attack vectors. Vendor compromise can provide direct access to client environments through existing trusted connections.
Contextual Example
An attacker compromises a vendor providing software updates, inserting malware into the supply chain that affects all clients receiving the updates.
VishingVoice phishing where attackers use phone calls to impersonate legitimate entities and steal personal information.A vishing call from "tech support" tricks victims into providing remote system access or personal details.
Definition
Social engineering conducted via voice calls. Attackers use pretexting, urgency, and authority to manipulate targets. Often targets elderly individuals or employees with access to financial systems.
Contextual Example
A vishing call from someone claiming to be from tech support could trick victims into providing access to their systems or personal details.
Voice Call Attack VectorPhone calls used to carry out attacks like vishing, where attackers pose as legitimate institutions to steal information.An attacker impersonates a customer service rep from a known tech company and tricks victims into providing sensitive information.
Definition
The broader category of phone-based attack vectors. Includes vishing, IVR spoofing, and call center social engineering. Caller ID spoofing makes legitimate organizations appear to be calling.
Contextual Example
An attacker calls a victim, impersonating a customer service representative from a known tech company, and tricks them into providing sensitive information under the guise of verifying their account.
Vulnerable SoftwareSoftware with known or unknown security flaws that attackers can exploit to gain unauthorized access.Attackers exploit an unpatched web browser vulnerability to execute malicious scripts and access user data.
Definition
Applications with security flaws — unpatched CVEs, insecure configurations, or design weaknesses — that can be exploited. Vulnerability scanning and patch management are the primary mitigations.
Contextual Example
Attackers target vulnerable software, such as an unpatched version of a web browser, exploiting its flaws to execute malicious scripts and gain access to user data.
Watering Hole AttackAttackers compromise a website frequently visited by the intended target to deliver malware to specific visitors.Attackers compromise an industry website to infect visitors from a specific target organization with tailored malware.
Definition
A targeted attack where adversaries compromise websites known to be frequented by their actual target. Rather than attacking the target directly, they poison a trusted site the target regularly visits.
Contextual Example
Attackers might conduct a watering hole attack by compromising an industry-related website, infecting visitors from a specific organization with malware tailored to exploit their systems.
Wired Attack VectorPhysical network cabling that, if unprotected, can be tapped for eavesdropping or data interception.An attacker taps an unsecured network cable in an office building to intercept sensitive data across the network.
Definition
Physical attacks on wired network infrastructure — tapping Ethernet cables, installing network taps, or accessing unlocked network closets. Physical access to cabling enables passive monitoring of all unencrypted traffic.
Contextual Example
An attacker taps into an unsecured network cable in an office building, allowing them to intercept sensitive data transmitted across the network.
Wireless Attack VectorCommunication via radio frequencies susceptible to eavesdropping, evil twin attacks, or Wi-Fi cracking.Attackers use a Wi-Fi sniffer to capture unencrypted data over an open wireless network, gaining credentials.
Definition
Attacks exploiting wireless communication protocols. Include Wi-Fi eavesdropping (open networks), evil twin access points, WPS brute force, deauthentication attacks, and rogue access points.
Contextual Example
Attackers use a Wi-Fi sniffer to capture unencrypted data transmitted over an open wireless network, gaining access to sensitive information such as login credentials.

Application SecurityVulnerabilities compromising the integrity and functionality of software applications.Memory injection or buffer overflow attacks can lead to data breaches or application crashes.
Definition
Security weaknesses in software applications arising from insecure coding practices, design flaws, or inadequate testing. Include injection flaws, authentication issues, and improper access control.
Contextual Example
Application-level attacks such as memory injection or buffer overflow can undermine system integrity, potentially leading to data breaches or application crashes.

Definition
Occurs when a program writes more data to a memory buffer than it can hold, overwriting adjacent memory. Can cause crashes or, if exploited precisely, allow an attacker to redirect execution to injected code.
Contextual Example
An attacker exploits a buffer overflow by feeding excessive data into an input field, causing the application to crash or execute malicious code.

Vulnerabilities arising from cloud-specific characteristics: shared responsibility misunderstandings, misconfigured storage buckets, insecure APIs, excessive permissions, and multi-tenant isolation failures.

Misconfigured cloud storage settings can expose sensitive data to unauthorized users, leading to potential breaches and compliance violations.
Cross-Site Scripting (XSS)A vulnerability where attackers inject malicious scripts into websites, executed by other users' browsers.An attacker embeds harmful JavaScript in a webpage; any visitor's browser executes it, potentially stealing their data.
Definition
Injects malicious client-side scripts into web pages viewed by other users. Types include reflected (URL-based), stored (persistent in database), and DOM-based XSS. Can steal session cookies, credentials, or redirect users.
Contextual Example
An attacker exploits XSS by embedding harmful JavaScript into a web page. The script executes in the browser of any user who visits the page, potentially stealing their data.
Cryptographic VulnerabilityVulnerabilities stemming from weaknesses in encryption methods or cryptographic protocols.Weak encryption keys or flawed algorithms make sensitive data easier for attackers to decrypt.
Definition
Security weaknesses in cryptographic implementations: use of broken algorithms (MD5, SHA-1, DES), short key lengths, poor random number generation, or improper protocol implementation.
Contextual Example
Flaws in cryptographic algorithms, such as weak encryption keys, can make sensitive data easier for attackers to decrypt, compromising confidentiality.
End-of-Life (EOL)Devices or software no longer receiving security updates, rendering them vulnerable to unpatched exploits.End-of-life devices are attractive targets because they lack security patches for known vulnerabilities.
Definition
The point at which a vendor ceases providing security updates and support for a product. All vulnerabilities discovered after EOL remain permanently unpatched, representing an ever-growing attack surface.
Contextual Example
End-of-life devices are susceptible to exploitation due to the lack of security patches and updates, making them attractive targets for attackers.
FirmwareLow-level software embedded in hardware devices that can be exploited if not regularly updated.Outdated router firmware is exploited to gain unauthorized network access, compromising all connected devices.
Definition
Software permanently programmed into hardware. Firmware vulnerabilities are particularly dangerous because they operate below the OS level, may persist through OS reinstalls, and are often difficult to update in embedded devices.
Contextual Example
An attacker exploits outdated firmware in a router to gain unauthorized access to the network, compromising all connected devices.
Hardware ProviderSuppliers of physical components that may introduce vulnerabilities through faulty or insecure hardware.An insecure hardware provider distributes compromised devices with backdoors enabling unauthorized access.
Definition
Hardware supply chain risks where components from untrusted providers contain intentional backdoors, counterfeit chips, or hardware-level trojans. Extremely difficult to detect and remediate.
Contextual Example
An insecure hardware provider may distribute compromised devices, enabling attackers to exploit the weaknesses in those devices for unauthorized access.
Hardware VulnerabilityVulnerabilities associated with physical devices, firmware, and hardware components.Firmware vulnerabilities in network routers expose systems to serious security risks allowing unauthorized control.
Definition
Security weaknesses in physical hardware — CPU design flaws (Spectre/Meltdown), BIOS/UEFI vulnerabilities, memory chips, and network interface cards. Often require firmware updates or hardware replacement to remediate.
Contextual Example
Firmware vulnerabilities in network routers or legacy hardware can expose systems to serious security risks, allowing attackers unauthorized control.
JailbreakingRemoving restrictions on a device's operating system to allow unauthorized software installation, increasing security risks.Jailbreaking an iPhone allows unauthorized apps, making the device more vulnerable to malware and exploits.
Definition
Exploiting vulnerabilities in iOS (or similar locked OS) to remove manufacturer restrictions, enabling unsigned software installation. Bypasses security controls including sandboxing and code signing verification.
Contextual Example
Jailbreaking an iPhone allows users to install unauthorized apps, making the device more vulnerable to malware and other security threats.
LegacyOutdated hardware or software still in use but no longer supported, posing significant security risks.Running legacy systems like Windows XP increases exploitation risk from numerous unpatched vulnerabilities.
Definition
Systems that have outlived their supported lifecycle. Legacy software and hardware cannot be patched for new vulnerabilities, creating permanent security gaps that require compensating controls or replacement.
Contextual Example
Running legacy systems like Windows XP significantly increases the risk of exploitation due to unpatched vulnerabilities that attackers can easily target.
Malicious UpdateA security threat where legitimate software updates are tampered with to introduce malicious code.A compromised software update introduces backdoors into a previously secure system during routine patching.
Definition
Supply chain attacks where software update mechanisms are compromised to distribute malware. The SolarWinds Orion compromise is the most notable example — affecting 18,000+ organizations through a poisoned update.
Contextual Example
During routine patching, a compromised software update introduces backdoors into a previously secure system, allowing attackers ongoing access.
Memory InjectionAn attack technique where malicious code is inserted into a program's memory to alter its behavior.An attacker uses memory injection to introduce harmful code into an application, causing unauthorized access.
Definition
Techniques like DLL injection, process hollowing, and reflective loading insert malicious code into legitimate running processes. The malicious code runs under the context of the legitimate process, evading detection.
Contextual Example
An attacker uses memory injection to introduce harmful code into an application, resulting in unauthorized access or application crashes.
MisconfigurationIncorrect settings in software, networks, or devices that expose them to potential exploitation.A firewall misconfiguration leaves sensitive data accessible to unauthorized users, causing a compliance breach.
Definition
One of the most common vulnerability categories. Includes default credentials left unchanged, unnecessary services enabled, overly permissive access controls, and improperly configured cloud storage. Often the root cause of major breaches.
Contextual Example
A firewall misconfiguration leaves sensitive data accessible to unauthorized users, potentially leading to breaches and compliance issues.
Mobile DeviceSecurity risks specific to smartphones and tablets, including app vulnerabilities and insecure settings.Mobile device vulnerabilities such as malicious apps can expose sensitive personal data to unauthorized access.
Definition
Mobile-specific vulnerabilities include jailbreaking/rooting, malicious apps, insecure Wi-Fi connections, SMS phishing, and weak device encryption. MDM (Mobile Device Management) is a key mitigation.
Contextual Example
Mobile device vulnerabilities, such as jailbreaking or installing malicious apps, can expose sensitive personal data to unauthorized access or theft.
OS-Based VulnerabilityVulnerabilities specific to the operating system that potentially compromise the system's overall security.Attackers target unpatched OS vulnerabilities in Windows or Linux to escalate privileges and deepen access.
Definition
Security flaws in the operating system kernel, system services, or bundled applications. OS vulnerabilities can allow privilege escalation, remote code execution, or denial of service attacks.
Contextual Example
Attackers often target unpatched OS vulnerabilities in Windows or Linux to escalate their privileges and gain deeper access to the system.
Race ConditionsVulnerabilities arising when system behavior depends on the timing or sequence of events.An attacker manipulates timing in a multi-threaded application to bypass security checks and gain unauthorized access.
Definition
Occur when multiple processes access shared resources concurrently without proper synchronization. The outcome depends on which process runs first, creating a window of opportunity for exploitation.
Contextual Example
An attacker manipulates the timing of events in a multi-threaded application, exploiting race conditions to bypass security checks and gain unauthorized access.
Resource ReuseUsing resources in virtual environments without proper sanitization can expose sensitive data.Without proper sanitization, data from one VM session could be accessed by the next VM user.
Definition
In cloud and virtual environments, storage and memory allocated to one tenant may contain residual data from a previous tenant if not properly zeroed/sanitized before reallocation.
Contextual Example
If resource reuse is not managed correctly, sensitive data from one virtual machine session could be accessed by another user, leading to data leaks.
Service Provider VulnerabilityA third-party IT service provider can introduce vulnerabilities if their security is compromised.A compromised service provider creates widespread security risks across their entire client base.
Definition
Vulnerabilities introduced through third-party service relationships. Service providers often have privileged access to client environments, making their compromise a high-impact event for all their clients.
Contextual Example
A compromised service provider can lead to widespread security risks across their client base, potentially exposing sensitive data across multiple organizations.
Side LoadingInstalling apps from unofficial sources, bypassing official app stores, increasing security risks.Users who side-load Android apps from untrusted sources may inadvertently install malware.
Definition
Installing applications from sources other than the official app store, bypassing security vetting. Sideloaded apps bypass code signing verification and app store security reviews, potentially containing malware.
Contextual Example
Users who use side-loading apps on Android devices may inadvertently install malware from untrusted sources, compromising device security.
Software ProviderThird-party software companies can introduce vulnerabilities if their products are compromised.A software provider could distribute malware-infected updates, causing breaches across their user base.
Definition
Vulnerabilities introduced through third-party software dependencies. Include compromised update servers, typosquatting package names in repositories, and abandoned libraries with known vulnerabilities.
Contextual Example
A software provider could inadvertently distribute malware-infected updates to clients, resulting in widespread security breaches across their user base.
SQL Injection (SQLi)An attack manipulating database queries through unsanitized user input, leading to data exposure or modification.An attacker sends malicious SQL commands to a web application, exposing sensitive customer data from the database.
Definition
Injects malicious SQL code through user input fields into database queries. Can extract confidential data, modify database records, bypass authentication, or execute OS commands. Prevention requires parameterized queries.
Contextual Example
During a SQL injection attack, an attacker sends malicious SQL commands to a web application, potentially exposing sensitive customer data stored in the database.
Supply Chain VulnerabilityVulnerabilities arising from third-party hardware, software, or service providers within a company's supply chain.Supply chain attacks compromise third-party software updates, introducing malicious code affecting end customers.
Definition
Security weaknesses introduced through the extended network of suppliers and partners. Any trusted third-party component — hardware, software, or service — can become a vector for attacking the primary organization.
Contextual Example
Supply chain attacks compromise third-party software updates, potentially introducing malicious code that affects the end customer's security.
Time-of-Check (TOC)A race condition where a vulnerability exists between verifying a resource's state and using it.An attacker alters a file after validity is checked but before it is used, leading to unauthorized actions.
Definition
The "check" phase of a TOCTOU race condition. An attacker changes the state of a resource between when it's verified (checked) and when the result is acted upon, exploiting the time gap between verification and use.
Contextual Example
An attacker takes advantage of a TOC vulnerability by altering a file/resource after it has been checked for validity but before it is used, leading to unauthorized actions.
Time-of-Use (TOU)A race condition vulnerability arising between the validation and execution of a resource.An attacker modifies an authenticated file immediately before it is executed, compromising the application's operation.
Definition
The "use" phase of a TOCTOU vulnerability. Between when a resource is validated and when it's actually used, an attacker replaces it with a malicious version. Together TOC and TOU form the TOCTOU attack class.
Contextual Example
In a TOU attack, an attacker modifies an authenticated file or resource immediately before it is executed, compromising the application's intended operation.
Virtual Machine (VM) EscapeAn attack where malware escapes from a virtual machine to affect the host system or other VMs.A VM escape vulnerability allows an attacker to move from a compromised VM to the host system.
Definition
Exploits hypervisor vulnerabilities to break out of a VM's isolation boundary, gaining code execution on the host or other VMs. Considered a critical severity vulnerability in virtualized environments.
Contextual Example
A VM escape vulnerability allows an attacker to move from a compromised virtual environment to the host system, compromising sensitive data and applications.
VirtualizationSecurity risks associated with virtual environments, including VMs and hypervisors.VM escape occurs when an attacker breaks out of a VM to compromise the underlying host system.
Definition
Virtualization introduces unique attack surfaces: hypervisor vulnerabilities, VM sprawl, snapshot exposure, and resource sharing between tenants. Hypervisor compromise gives attackers access to all VMs it manages.
Contextual Example
VM escape occurs when a malicious actor successfully breaks out of a virtual machine to compromise the underlying host system, leading to broader system vulnerabilities.
Web-Based VulnerabilitySecurity threats targeting websites and web applications through malicious input or design flaws.SQL injection and XSS are common web-based vulnerabilities leading to unauthorized data access or manipulation.
Definition
Vulnerabilities in web applications and sites. OWASP Top 10 catalogs the most critical: injection, broken access control, cryptographic failures, XSS, and security misconfigurations are consistently highest risk.
Contextual Example
SQL injection (SQLi) and cross-site scripting (XSS) are common web-based vulnerabilities that can lead to unauthorized data access or manipulation.
Zero-DayA vulnerability unknown to the software vendor that is actively exploited before a fix is available.Zero-day vulnerabilities allow attackers to exploit systems without fear of detection or patching, causing widespread damage.
Definition
A vulnerability unknown to the vendor and/or without an available patch. Attackers who discover or purchase zero-days can exploit them indefinitely until disclosed and patched. Highly valued in criminal and nation-state markets.
Contextual Example
Zero-day vulnerabilities pose a significant threat, as attackers can exploit them without fear of immediate detection or patching, leading to potential widespread damage.

Account LockoutA condition triggered when multiple failed login attempts are detected, potentially indicating a brute force attack.Multiple failed login attempts leading to account lockout signal a possible brute-force attack to the security team.
Definition
A detective control triggering when a defined number of failed authentication attempts occur within a time window. Protects against brute force but can also be used as a DoS attack against user accounts.

Multiple failed login attempts led to an account lockout, signaling a possible brute-force attack on the security team.

Definition
Sends spoofed requests to vulnerable reflectors (DNS, NTP, memcached) using the victim's IP as the source. The reflectors send large responses to the victim, amplifying traffic by factors of 100x or more.

Attackers launch an amplified DDoS attack by exploiting publicly accessible DNS servers to send overwhelming traffic to the victim's server, disrupting services.
Birthday AttackA cryptographic attack exploiting probability mathematics to find two different inputs with the same hash value.A birthday attack enables an attacker to discover two messages with the same hash, breaking digital signature security.
Definition
Based on the birthday paradox in probability. Exploits that hash collisions occur more easily than intuition suggests. Used to forge digital signatures or create malicious documents with the same hash as legitimate ones.
Contextual Example
The birthday attack enables an attacker to discover two distinct messages that yield the same hash value, breaking the security of digital signatures.
BloatwareUnwanted software that takes up system resources, slows performance, and may include security risks.Pre-installed bloatware on a new laptop reduces processing speed and may include security vulnerabilities.
Definition
Pre-installed or bundled software consuming resources without providing significant value. Security risk because bloatware may have vulnerabilities, make unauthorized network connections, or be difficult to remove completely.
Contextual Example
A new laptop is delivered with pre-installed bloatware, which leads to noticeable reductions in processing speed due to unnecessary applications running in the background.
Brute ForceRepeatedly trying different combinations of passwords or keys to gain unauthorized access.A hacker systematically tries thousands of password combinations until successfully gaining admin account access.
Definition
Exhaustive search attack trying all possible combinations. Pure brute force tries every possibility; dictionary attacks use word lists; credential stuffing uses known breached passwords. Mitigated by MFA, account lockout, and strong passwords.
Contextual Example
A hacker employs a brute force attack on the admin account, systematically trying thousands of password combinations until they successfully gain access.
Collision AttackAn attack where two different inputs produce the same cryptographic hash, compromising data integrity.A collision attack exploits hashing algorithm weaknesses, causing two files to generate the same hash value.
Definition
Finds two distinct inputs that produce an identical hash output. Undermines hash functions used for integrity verification. MD5 and SHA-1 are both vulnerable to practical collision attacks.
Contextual Example
A collision attack exploits weaknesses in a hashing algorithm, causing two files to generate the same hash value and thereby threatening data integrity.
Credential ReplayReusing stolen authentication credentials or tokens to gain unauthorized access to systems.An attacker uses captured credentials to log into a corporate VPN, bypassing authentication controls.
Definition
Captures valid authentication tokens or credentials and retransmits them to gain unauthorized access. Mitigated by using session tokens with expiry, TLS for all authentication traffic, and challenge-response protocols.
Contextual Example
An attacker can use credential replay to log into a corporate VPN using previously stolen username and password pairs, bypassing security measures.
Distributed Denial-of-Service (DDoS)An attack overwhelming a target system with excessive traffic from multiple sources, making it unavailable.Thousands of compromised devices flood a company's web server with traffic, crashing it and making it inaccessible.
Definition
Uses a botnet of compromised devices (zombies) to flood targets with traffic or resource-exhausting requests. Harder to mitigate than single-source DoS. Types include volumetric (bandwidth), protocol, and application-layer attacks.
Contextual Example
The company's website suffers a DDoS attack. Thousands of compromised devices flood the server, crashing it and making it inaccessible to users.
