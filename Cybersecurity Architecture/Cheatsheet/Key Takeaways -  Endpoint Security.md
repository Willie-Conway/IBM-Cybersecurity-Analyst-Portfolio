# Key Takeaways: Endpoint Security

**Estimated time needed: 5 minutes**

## Core Concepts

- **Multi-factor authentication** is based on trust in the source platform
- **Endpoints** refer to physical devices that connect to a computer network for information exchange
- **Vulnerability** can come from both hardware and software devices

## Visibility and Control

**Visibility and control** are keys to security. They dictate the type of hardware and software that are allowed within an organization.

### Forms of Control

| Control Type                | Description                                                |
| --------------------------- | ---------------------------------------------------------- |
| **Inventory**         | Track all endpoint devices in the organization             |
| **Policies**          | Establish rules for secure endpoint usage                  |
| **Patching**          | Keep software and firmware up to date                      |
| **Encryption**        | Protect data at rest and in transit                        |
| **Remote wipe**       | Securely erase data from lost or stolen devices            |
| **Location tracking** | Monitor device location for asset management and security  |
| **Anti-virus**        | Detect and block malicious software                        |
| **Disposal**          | Securely destroy or sanitize data before device retirement |

---

## Policies

Policies are implemented to achieve security. They include:

- **Password requirements** – certain length, strength, and expiry date
- **Encryption standards** – ensuring proper encryption is used

> Policies should also evaluate whether devices are monitored and under what conditions.
